 WHAT IS BIOECHO?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

BioEcho is a multimodal AI system that predicts 6 health biomarkers from a
60-second phone scan — using only your voice, camera, eye movement, and typing.
No wearable. No blood test. No clinic.

It builds a personal "Bio Signature" — a 256-dimensional vector that fingerprints
your unique health state. Tracked weekly, it detects health deterioration up to
6 months before clinical symptoms appear.

Pitch line:
"A blood test costs ₹500 and requires a clinic.
 BioEcho costs nothing, requires only your phone, and happens in 60 seconds."

 SYSTEM ARCHITECTURE — 4 MODALITIES → FUSION → 6 TARGETS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

INPUT MODALITIES:
┌─────────────────────────────────────────────────────────────────────────────┐
│ 1. VOICE (60-second audio)                                                  │
│    → HandCrafted: MFCC, Jitter, Shimmer, HNR, Spectral, F0, Tempo          │
│    → Deep:        Wav2Vec2 (768-dim contextual speech embedding)            │
│    → Detects:     Parkinson's tremor, Depression, Diabetes, Stress          │
│    → Research:    Little et al. 2009, Williamson 2016, Quatieri 2012        │
├─────────────────────────────────────────────────────────────────────────────┤
│ 2. FACE (60-second video — phone front camera)                              │
│    → rPPG via CHROM algorithm on 4 facial ROIs (forehead, cheeks, nose)    │
│    → Extracts:    Heart Rate, HRV (RMSSD, SDNN, pNN50)                     │
│    → Detects:     Cardiac stress, Anaemia, Blood pressure proxy             │
│    → Research:    de Haan & Jeanne 2013, Chen & McDuff 2018                 │
├─────────────────────────────────────────────────────────────────────────────┤
│ 3. EYE TRACKING (from same video — MediaPipe iris)                          │
│    → Gaze position, velocity, saccade analysis, pupil diameter, blinks     │
│    → Detects:     Parkinson's (slow saccades), Alzheimer's (latency),       │
│                   Cognitive load (pupil dilation), ADHD (fixation)          │
│    → Research:    Leigh & Zee 2015, Mosimann 2005, Kahneman 1973           │
├─────────────────────────────────────────────────────────────────────────────┤
│ 4. TYPING DYNAMICS (background keyboard logging during scan)                │
│    → Dwell time, flight time, digraph latency, error rate, rhythm           │
│    → Detects:     Motor decline, Cognitive load, Stress, Tremor             │
│    → Research:    Giancardo 2016, Kautzky-Willer 2021                      │
└─────────────────────────────────────────────────────────────────────────────┘

FUSION:
  CrossModalTransformer
  → Treats 4 modality embeddings as a 4-token sequence
  → Self-attention: each modality attends to all others
  → Modal dropout: randomly drops a modality during training (robustness)
  → Output: 256-dim Bio Signature vector

OUTPUT TARGETS:
  1. heart_rate      — beats per minute
  2. hrv_rmssd       — heart rate variability (ms)
  3. stress_score    — 0-100 proxy score
  4. cognitive_load  — 0-100 mental workload index
  5. neuro_risk      — 0-100 neurological risk score
  6. bio_score       — 0-100 overall health trajectory (the main metric)

 MODEL ARCHITECTURE DETAILS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

AudioEncoder:
  - MLP on 200 handcrafted features → 256-dim
  - Linear projection of 768-dim Wav2Vec2 → 256-dim
  - Learned fusion gate (softmax over 2 sources)
  - Output: (B, 256)

RPPGEncoder:
  - 3-branch multi-scale 1D CNN (kernel sizes 3, 7, 15)
  - Each branch: Conv → GELU → AdaptivePool → Conv → Pool
  - Concatenate 3 branches → flatten → Linear → 256-dim
  - HR/HRV features (4-dim) encoded separately → 256-dim
  - Gated fusion → Output: (B, 256)

GazeEncoder:
  - Bidirectional LSTM (3 layers, hidden=128 → 256-dim with bidir)
  - Temporal attention: learns which timesteps matter most
  - Output: attention-weighted context vector (B, 256)

KeystrokeEncoder:
  - Linear projection: 5-dim → 256-dim
  - Learned positional embedding
  - [CLS] token prepended
  - Transformer Encoder (4 heads, 3 layers, GELU)
  - [CLS] token output → (B, 256)

CrossModalTransformer:
  - 4 modality tokens + 1 [FUSION] token → 5-token sequence
  - Modality type embeddings (like token types in BERT)
  - Pre-norm TransformerEncoder (6 layers, 8 heads, FF=1024)
  - Modal dropout: randomly zeros a modality token at 15% rate
  - [FUSION] token → (B, 256) final bio-signature

Prediction Heads (×6):
  - Linear(256→128) → GELU → Dropout → Linear(128→64) → GELU → Linear(64→1)

Uncertainty Heads (×6):
  - Linear(256→64) → GELU → Linear(64→1) → Softplus
  - Predicts aleatoric uncertainty (variance) per target

Total parameters: ~15-20M (fits comfortably on 2x T4)

 TRAINING CONFIGURATION
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Hardware:      Kaggle 2x NVIDIA T4 GPU (DataParallel)
Precision:     Mixed (AMP, fp16 compute / fp32 accumulation)
Batch size:    16 (8 per GPU × 2 GPUs)
Epochs:        50 (with early stopping, patience=10)
Optimizer:     AdamW (differential LRs)
               - Encoders:   LR × 0.3
               - Fusion:     LR × 1.0
               - Heads:      LR × 2.0
LR:            2e-4 base
Schedule:      Linear warmup (5 epochs) + cosine annealing
Weight decay:  1e-4
Grad clipping: 1.0
Loss:          Gaussian NLL with learned per-target uncertainty
               + per-target weights (neuro_risk × 2.0, bio_score × 2.0)

 KEY TECHNICAL DECISIONS & WHY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

WHY GAUSSIAN NLL loss (not MSE)?
  MSE forces the model to commit to point predictions.
  Gaussian NLL trains the model to say "I predict 78 bpm, but I'm uncertain
  (±12 bpm)." In healthcare, knowing when the model is unsure is critical.
  Overconfident wrong predictions in healthcare = dangerous.

WHY modal dropout?
  In real deployment, sometimes the camera fails, or background noise ruins
  the audio. Modal dropout (randomly zeroing a modality during training)
  forces the fusion transformer to never rely 100% on any single signal.
  Result: BioEcho still works even with only 2 of 4 modalities.

WHY Mahalanobis distance for drift?
  Euclidean distance treats all 256 embedding dimensions equally.
  Mahalanobis uses the inverse covariance matrix — it knows which directions
  of the embedding space have high vs low variance in YOUR baseline.
  A change in a stable dimension matters more than a change in a noisy one.
  This is why it outperforms simple threshold alerts.

WHY Wav2Vec2 + handcrafted (not just Wav2Vec2)?
  Wav2Vec2 learns rich contextual representations but can miss specific
  low-level biomarker signals (jitter, shimmer) that are clinically validated.
  The gated fusion lets the model learn when to trust the deep embedding vs
  the engineered feature — giving best of both worlds.
